# Overview

**Pipeline Vision** is a real-time analytics dashboard I built to solve a critical problem in biotech investing: **tracking hundreds of clinical programs across dozens of companies is humanly impossible**, yet missing a single Phase 2 readout can mean losing a 50-100% return opportunity.

Traditional approaches rely on:
- Manual Excel trackers (quickly outdated)
- Quarterly earnings calls (always lagging)
- Sell-side research (biased and incomplete)

I built Pipeline Vision to automatically track every meaningful clinical catalyst across the biotech sector and calculate probability-adjusted valuations in real-time.

---

# The Problem

## Information Overload

**Typical biotech investor challenge:**
- 500+ public biotech companies
- Each with 3-10 pipeline assets
- Clinical trials take 2-8 years to read out
- Catalysts announced with little notice (FDA approvals, trial data, partnership deals)

**Reality:** Impossible to manually track 2,000+ pipeline assets and react quickly to new data.

## Valuation Complexity

**Traditional DCF models for biotech are tedious:**
- Need to model each pipeline asset separately
- Probability of success varies by phase (Phase 1: 10%, Phase 2: 30%, Phase 3: 60%)
- Peak sales estimates require market research
- Adjusting models after every data point is time-consuming

**Result:** Most investors use crude heuristics instead of rigorous valuation.

# What I Built

## Core Features

### 1. Automated Pipeline Tracker

Monitors 500+ biotech companies in real-time:

**Data Sources:**
- ClinicalTrials.gov API → Trial start dates, enrollment, estimated completion
- SEC filings (10-K, 10-Q) → Pipeline updates, trial costs
- Company press releases → Data readouts, regulatory updates
- FDA PDUFA calendar → Approval decision dates
- Patent expiration databases → LOE timelines

**Auto-generated alerts for:**
- New trial initiations
- Data readout announcements
- FDA approval decisions
- Partnership deals
- Pipeline discontinuations

### 2. Probability-Weighted NPV Calculator

For each pipeline asset, automatically calculates:

**Clinical Success Probabilities:**
- Uses historical biotech phase transition rates
- Adjusts for indication (oncology vs rare disease vs CNS)
- Incorporates trial design factors (adaptive design, biomarker selection)
- Updates probabilities based on interim data

**Peak Sales Modeling:**
- Addressable patient population (epidemiology databases)
- Pricing assumptions based on comp set
- Market share projections with competitive dynamics
- Time to peak sales (indication-specific)

**rNPV Calculation:**
```
rNPV = Σ (Probability × Peak Sales × Market Share × NPV Multiple) - Dev Costs
```

### 3. Interactive Visualizations

Built custom dashboards with **Plotly** and **React**:

**Portfolio View:**
- Heatmap of all companies by clinical stage and indication
- Bubble chart: Market cap vs rNPV (find undervalued assets)
- Timeline view: Upcoming catalysts over next 24 months

**Company Deep Dive:**
- Waterfall chart: rNPV buildup by asset
- Sensitivity analysis: How rNPV changes with assumptions
- Catalyst calendar: Trial readouts, FDA dates, partnership timelines
- Comp table: Peer valuations by clinical stage

**Sector Analytics:**
- Which indications are most crowded?
- Historical success rates by phase and indication
- Aggregate biotech sector rNPV vs market cap

# Technical Implementation

## Architecture

### Backend Data Pipeline

**Daily ETL Process (Apache Airflow):**

```python
# Pseudocode for pipeline orchestration
DAG: daily_pipeline_update
  ├─ Task 1: Scrape ClinicalTrials.gov (new trials, status changes)
  ├─ Task 2: Parse SEC filings (pipeline updates)
  ├─ Task 3: Fetch FDA calendar (PDUFA dates)
  ├─ Task 4: Update clinical probabilities (new data)
  ├─ Task 5: Recalculate rNPV for affected assets
  └─ Task 6: Generate alerts (material changes)
```

**Data Storage (PostgreSQL):**
- `companies` table → Company metadata, market cap, cash
- `pipeline_assets` table → Indication, phase, target, mechanism
- `clinical_trials` table → NCT ID, design, endpoints, timeline
- `catalysts` table → Upcoming events, dates, asset mapping
- `rnpv_snapshots` table → Historical valuations over time

### API Layer (FastAPI)

RESTful API serving data to frontend:

```python
GET /companies → List all companies with pipeline counts
GET /companies/{ticker}/pipeline → Detailed assets for one company
GET /companies/{ticker}/rnpv → Current rNPV calculation with breakdown
GET /catalysts/upcoming → Next 30-60-90 day catalyst calendar
GET /sector/analytics → Aggregate sector metrics
POST /models/rnpv → Custom rNPV calculation with user assumptions
```

### Frontend (React + Plotly)

**Tech Stack:**
- React 18 with hooks
- Plotly.js for interactive charts
- Tailwind CSS for styling
- React Query for data fetching and caching
- Zustand for state management

**Key Components:**
- `<PipelineTable>` → Sortable, filterable table with 20+ columns
- `<CatalystCalendar>` → Timeline view with drill-down
- `<rNPVWaterfall>` → Interactive chart showing value buildup
- `<SensitivityHeatmap>` → 2D grid: success prob vs peak sales
- `<ComparisonMatrix>` → Peer valuation benchmarking

# Key Insights & Findings

## Insight 1: Market Inefficiency at Phase Transitions

**Discovery:** Stock prices lag clinical data by 2-10 days for small/mid-cap biotech

**Evidence:**
- Tracked 50 Phase 2 readouts in 2024
- Compared immediate market reaction (+/- 1 day) vs 10-day return
- Found 40% of positive readouts continued to rally for 5-7 days

**Investment Implication:** Systematic screening for positive data + quick position entry can capture alpha

## Insight 2: rNPV Multiple Compression

**Discovery:** Market values clinical-stage assets at 30-50% discount to probability-adjusted NPV

**Evidence:**
- Calculated rNPV for 100 biotech companies
- Compared to market cap (adjusting for cash)
- Median: Market cap = 0.45× rNPV

**Investment Implication:** Market applies risk premium beyond clinical probabilities - buying Phase 2/3 assets at 30-40% of rNPV can be attractive

## Insight 3: Indication Matters More Than Phase

**Discovery:** Oncology assets trade at premium multiples vs CNS, even at earlier stages

**Evidence:**
- Phase 2 oncology: Market cap / rNPV = 0.6×
- Phase 2 CNS: Market cap / rNPV = 0.3×

**Investment Implication:** Indication-specific risk premiums are persistent - CNS offers better risk/reward for risk-tolerant investors

# Technical Challenges & Solutions

## Challenge 1: Data Quality

**Problem:** ClinicalTrials.gov often outdated, companies don't update trial status

**Solutions:**
- Cross-reference multiple sources (SEC filings + company IR sites)
- Flag discrepancies for manual review
- Add "last updated" timestamps and data freshness indicators
- Built manual override system for known inaccuracies

## Challenge 2: Peak Sales Estimation

**Problem:** Peak sales is subjective and drives 80% of rNPV variance

**Solutions:**
- Used multiple methods:
  - **Epidemiology-based:** Patient population × penetration × price
  - **Analog-based:** Comparable approved drugs in indication
  - **Analyst consensus:** Average of sell-side estimates
- Display range (low/base/high) instead of point estimate
- Sensitivity analysis showing rNPV at different peak sales levels

## Challenge 3: Performance at Scale

**Problem:** Recalculating rNPV for 500 companies × 5 assets each = 2,500 calculations

**Solutions:**
- **Incremental updates:** Only recalc when input data changes
- **Caching:** Store computed rNPV, invalidate on data change
- **Background jobs:** Heavy calculations run async, UI shows cached values
- **Database indexing:** Optimized queries on pipeline_assets table

**Result:** Dashboard loads in &lt;2 seconds, updates in &lt;500ms

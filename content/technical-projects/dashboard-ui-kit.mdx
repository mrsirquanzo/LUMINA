# Overview

Building **Pipeline Vision** and **BioScout** taught me that biotech analytics tools have unique UI requirements: dense data displays, complex filtering, scientific visualizations, and real-time updates. After designing ad-hoc components for each project, I built a comprehensive UI component library to:

- **Standardize UX patterns** across biotech data applications
- **Speed up development** with ready-to-use, tested components
- **Ensure accessibility** for data-heavy interfaces
- **Maintain visual consistency** across multiple tools

This UI kit includes 80+ components specifically designed for biotech analytics, from clinical trial cards to rNPV calculators to molecular structure viewers.

# Design Philosophy

## Principles for Data-Heavy Biotech UIs

### 1. Information Density vs Clarity
**Challenge:** Investors need to see lots of data, but cluttered UIs are unusable
**Solution:** Progressive disclosure - show summary, allow drill-down

**Example:** Pipeline asset card
- **Level 1 (Card):** Indication, phase, next milestone
- **Level 2 (Expanded):** Trial design, endpoints, timeline
- **Level 3 (Full page):** Complete trial protocol, competitive context

### 2. Scientific Accuracy
**Challenge:** Oversimplified UIs can misrepresent complex biology
**Solution:** Design for precision - include units, error bars, confidence levels

**Examples:**
- Always show units (mg/kg, % change, p-value)
- Display confidence intervals, not just point estimates
- Allow hovering for full precision (show "1.234" not "1.2")

### 3. Scannable Data
**Challenge:** Users need to quickly find signal in noisy biotech data
**Solution:** Visual encoding (color, icons, typography) to highlight key information

**Visual hierarchy:**
- **Critical data** → Large, bold, color-coded (e.g., FDA approval in green)
- **Important data** → Medium weight, normal text
- **Context data** → Small, gray text
- **Metadata** → Footnotes and tooltips

### 4. Multi-Dimensional Filtering
**Challenge:** Biotech data has many axes (phase, indication, company, date, etc.)
**Solution:** Compound filters with saved views

**Filter types:**
- **Categorical** - Checkboxes for phase, indication, company
- **Range** - Sliders for market cap, rNPV, timeline
- **Date** - Calendar pickers for trial start/end dates
- **Search** - Fuzzy search on company names, indications, targets

### PipelineAssetCard

**Purpose:** Display clinical asset overview

**Content:**
- Indication (e.g., "Uveal melanoma")
- Phase (e.g., "Phase 3")
- Target/mechanism (e.g., "TCR therapy targeting gp100")
- Next catalyst (e.g., "Phase 3 data - Q2 2025")

**Visual design:**
- Phase color-coded badge (top right)
- Large indication name (bold, 18pt)
- Timeline progress bar
- Status icon (✓ green, ⚠ yellow, ✗ red)

**Interactions:**
- Click to expand full details
- Hover shows quick stats tooltip
- Drag to reorder in priority view

### CompanyCard

**Purpose:** Company snapshot in portfolio view

**Content:**
- Company name and ticker
- Market cap
- Number of pipeline assets by phase
- Key partnerships
- Upcoming catalysts (count)

**Visual design:**
- Logo (if available)
- Mini bar chart (assets by phase)
- Metric grid (2×2 layout)
- Next catalyst badge

**Interactions:**
- Click to navigate to company detail page
- Star to add to watchlist
- Quick actions menu (3-dot icon)

### CatalystTimeline

**Purpose:** Visualize upcoming events over time

**Content:**
- Events plotted on horizontal timeline
- Event types: Data readouts, FDA decisions, earnings
- Grouping by company/indication

**Visual design:**
- Horizontal timeline (months across top)
- Events as colored dots/icons
- Leader lines to labels (avoid overlap)
- Filters on left (company, event type)

**Interactions:**
- Click event for details modal
- Hover shows event preview
- Zoom timeline (30/60/90/180 days)

### FinancialMetricsPanel

**Purpose:** Display key financial metrics

**Metrics shown:**
- Market cap
- Enterprise value
- Cash position and burn rate
- Runway (months)
- Revenue (TTM)

**Visual design:**
- Metrics in 2×3 grid
- Large number (primary value)
- Small label (metric name)
- Trend indicator (↑↓→)
- Sparkline chart (historical trend)

**Interactions:**
- Click metric to see historical chart

### rNPVWaterfallChart

**Interactions:**

### Example Component: PipelineAssetCard

```typescript
// PipelineAssetCard.tsx
import { FC } from 'react';

export interface PipelineAssetCardProps {
  indication: string;
  phase: 'Preclinical' | 'Phase 1' | 'Phase 2' | 'Phase 3' | 'Approved';
  target: string;
  mechanism: string;
  nextCatalyst: {
    description: string;
    date: string;
  };
  status: 'on-track' | 'at-risk' | 'delayed';
  onClick?: () => void;
}

export const PipelineAssetCard: FC<PipelineAssetCardProps> = ({
  indication,
  phase,
  target,
  mechanism,
  nextCatalyst,
  status,
  onClick
}) => {
  const phaseColor = {
    'Preclinical': 'bg-gray-100 text-gray-700',
    'Phase 1': 'bg-blue-100 text-blue-700',
    'Phase 2': 'bg-blue-200 text-blue-800',
    'Phase 3': 'bg-blue-300 text-blue-900',
    'Approved': 'bg-green-100 text-green-800'
  }[phase];

  const statusIcon = {
    'on-track': '✓',
    'at-risk': '⚠',
    'delayed': '✗'
  }[status];

  const statusColor = {
    'on-track': 'text-green-600',
    'at-risk': 'text-yellow-600',
    'delayed': 'text-red-600'
  }[status];

  return (
    <div
      onClick={onClick}
      className="card hover:shadow-lg transition-shadow cursor-pointer"
    >
      <div className="flex justify-between items-start mb-3">
        <h3 className="text-xl font-semibold text-gray-900">{indication}</h3>
        <span className={`px-3 py-1 rounded-full text-sm font-medium ${phaseColor}`}>
          {phase}
        </span>
      </div>

      <div className="text-sm text-gray-600 mb-4">
        <div><strong>Target:</strong> {target}</div>
        <div><strong>Mechanism:</strong> {mechanism}</div>
      </div>

      <div className="border-t border-gray-200 pt-3 flex justify-between items-center">
        <div>
          <div className="text-xs text-gray-500">Next Catalyst</div>
          <div className="text-sm font-medium text-gray-900">
            {nextCatalyst.description}
          </div>
          <div className="text-xs text-gray-500">{nextCatalyst.date}</div>
        </div>
        <div className={`text-2xl ${statusColor}`}>
          {statusIcon}
        </div>
      </div>
    </div>
  );
};
```

# Design Patterns & Best Practices

## Data-Heavy Tables

**Problem:** 20+ column tables are unreadable
**Solution:**
- Show 5-7 most important columns by default
- "Customize columns" menu to add/remove
- Save column preferences per user

**Problem:** Long tables with hundreds of rows
**Solution:**
- Virtual scrolling (render only visible rows)
- Pagination as fallback
- "Load more" button for mobile

## Real-Time Updates

**Pattern:** Optimistic UI updates
- Show update immediately
- Background sync to server
- Revert + show error if sync fails

**Visual design:**
- Pulsing indicator for "syncing"
- Toast notification for "saved"
- Inline error for "failed to save"

## Empty States

**Don't:** Show blank screen with generic "No data"
**Do:** Show helpful empty state with CTA

**Example: No pipeline assets**
```
[Icon: Empty folder]
No pipeline assets yet

Get started by adding your first clinical asset
or import from ClinicalTrials.gov

[Button: Add Asset]  [Button: Import]
```

## Loading States

**Skeleton screens > spinners**
- Show layout with gray blocks
- Maintains perceived performance
- Less jarring than spinner

**Progressive loading:**
- Load critical data first (above fold)
- Lazy load charts and heavy components
- Show "Loading..." only for slow requests (>2s)

# Accessibility

## WCAG 2.1 AA Compliance

**Color Contrast:**
- All text meets 4.5:1 contrast minimum
- Large text (18pt+) meets 3:1
- Tested with Stark plugin in Figma

**Keyboard Navigation:**
- All interactive elements focusable
- Visible focus indicators
- Logical tab order

**Screen Reader Support:**
- ARIA labels on interactive elements
- Landmarks for major sections
- Alt text for icons

**Data Table Accessibility:**
- `<th>` for headers with scope
- Caption describing table
- Sortable columns announced

## Color-Blind Friendly

**Don't rely on color alone:**
- Add icons to status (not just red/yellow/green)
- Use patterns in charts (stripes, dots)
- Include text labels

**Color palette:**
- Tested with colorblind simulators
- Distinguishable for most common types (deuteranopia, protanopia)

# Lessons Learned

## Design Process

**1. Design with Real Data**
- Early mockups used fake data → missed edge cases
- Redesigned tables 3 times when real data was longer/messier
- **Lesson:** Use production data samples from day 1

**2. Start with Mobile**
- Desktop-first designs don't adapt well to mobile
- Responsive biotech dashboards are hard (so much data!)
- **Lesson:** Design mobile view first, scale up

**3. Test with Users**
- Showed designs to 5 analysts before building
- Discovered 40% of features weren't needed
- **Lesson:** User testing saves development time

## Technical Challenges

**Challenge:** Figma components don't map 1:1 to React
**Solution:** Accepted some divergence, prioritized developer experience

**Challenge:** Tailwind + custom design tokens = maintenance burden
**Solution:** Used CSS variables for design tokens, Tailwind for utilities

**Challenge:** Storybook is slow with 80+ components
**Solution:** Lazy loading, component code splitting

## Collaboration

**Challenge:** Designers change Figma, developers don't notice
**Solution:** Figma webhooks → Slack notifications for component updates

**Challenge:** Figma designs diverge from production code
**Solution:** Screenshot testing (Chromatic) to catch regressions

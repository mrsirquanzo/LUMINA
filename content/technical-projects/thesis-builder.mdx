# Overview

**ThesisBuilder** is an AI-powered system that generates investment memos automatically, transforming what used to take 2-3 days of work into a 45-minute process. It combines web scraping, financial data APIs, scientific literature analysis, and document generation to produce professional-grade investment analyses.

## The Spark

After writing my third 25-page investment memo by hand (Absci, Immunocore, multispecific antibodies), I noticed a pattern: **the structure was always the same, and 60% of the content was information synthesis rather than original insights.**

**Standard memo structure:**
1. Executive Summary
2. Company Overview & History
3. Technical Deep Dive
4. Market Analysis
5. Competitive Landscape
6. Financial Assessment
7. Risk Analysis
8. Investment Recommendation

**The insight:** Steps 2, 4, 5, 6 are mostly information gathering and synthesis. AI can do this. Steps 3, 7, 8 require judgment - but AI can provide first drafts that I refine.

**The goal:** Automate the repetitive 60%, so I can spend my time on the high-value 40% (technical assessment and investment decision).

### 2. Multi-Agent LLM System

**Architecture:**
```python
class InvestmentAnalysisAgent:
    def __init__(self, role: str, tools: List[Tool]):
        self.role = role  # "technology", "market", "financial", etc
        self.llm = ChatOpenAI(model="gpt-4-turbo", temperature=0.1)
        self.tools = tools  # Specialized tools for each agent
        self.memory = ConversationBufferMemory()

    async def analyze(self, context: Dict) -> Analysis:
        # Agent-specific prompts with domain expertise
        prompt = self.build_prompt(context)
        response = await self.llm.agenerate(prompt, tools=self.tools)
        return self.structure_output(response)
```

**Why multi-agent vs single LLM:**
- Specialization: Each agent has domain-specific prompts and tools
- Quality: Focused agents produce more accurate analysis
- Debuggability: Can isolate and fix issues in specific sections
- Iterative refinement: Agents can critique each other's output

### 3. Financial Modeling

```python
class rNPVModel:
    def build_model(self, company_data: Dict) -> Valuation:
        # Extract pipeline from clinical trials data
        pipeline = self.extract_pipeline(company_data['trials'])

        # Estimate peak sales for each asset
        for asset in pipeline:
            asset.peak_sales = self.estimate_peak_sales(
                indication=asset.indication,
                prevalence=asset.patient_population,
                pricing=self.get_pricing_comps(asset.indication),
                market_share=self.estimate_share(asset.competitive_position)
            )

            # Probability of success by phase
            asset.pos = self.get_historical_pos(
                phase=asset.phase,
                indication=asset.indication,
                trial_design=asset.design
            )

            # NPV calculation
            asset.rnpv = self.calculate_npv(
                peak_sales=asset.peak_sales,
                probability=asset.pos,
                development_costs=self.estimate_dev_costs(asset.phase),
                discount_rate=0.12
            )

        # Aggregate company rNPV
        return sum(asset.rnpv for asset in pipeline)
```

# Technical Challenges

## Challenge 1: Information Synthesis

**Problem:** LLMs tend to summarize verbatim rather than synthesize insights

**Solution:**
- Engineered prompts with explicit synthesis instructions
- "Don't just list facts - explain what they mean for investors"
- Few-shot examples of good synthesis vs bad summary
- Iterative refinement: Agent 5 synthesizes output from Agents 1-4

## Challenge 2: Financial Model Accuracy

**Problem:** Peak sales estimates vary wildly, garbage-in-garbage-out

**Solution:**
- Multiple estimation methods (epidemiology, analog comps, analyst consensus)
- Show range (low/base/high) instead of point estimate
- Sensitivity tables showing rNPV at different assumptions
- Flag assumptions that have high impact on valuation

## Challenge 3: Hallucination Prevention

**Problem:** LLMs hallucinate references, statistics, and company claims

**Solution:**
- **Retrieval-first architecture**: All claims must trace to source document
- **Confidence scoring**: LLM outputs probability for each statement
- **Cross-validation**: Check claims against multiple sources
- **Human review layer**: Flag low-confidence claims for verification

**Result:** Reduced hallucination rate from ~15% (naive prompting) to ~3% (with safeguards)

## Challenge 4: Document Formatting

**Problem:** LLMs produce markdown but investors want polished PDFs

**Solution:**
- Convert markdown → LaTeX with custom templates
- Generate charts separately with matplotlib
- Use LaTeX for professional typography
- Support multiple export formats (PDF, Word, HTML)

### Product

- **Perfect is the enemy of good** - users want fast 80% solution over slow 95% solution
- **Trust through transparency** - showing sources and confidence scores critical for adoption

### Domain

- **AI won't replace analysts** - but it changes the job from data gathering to judgment
- **The bottleneck shifts** - from "I don't have time to research" to "Which companies should I research?"
- **Compound AI is powerful** - combining retrieval + generation + tools + multi-agent = step-change
